Тест-кейси для User Story: Купівля підписки на платформі шерінгу відео
Тест-кейс: Пошук цікавого відео
•Кроки:
	1.Увійти в додаток.
	2.Перейти до головної сторінки або скористатись пошуком.
	3.Ввести назву або тему відео, яке цікавить.
	4.Переглянути результати пошуку.
•Очікуваний результат: Відображаються релевантні відео за запитом користувача.

Тест-кейс: Перегляд відео та підписка на канал
•Кроки:
	1.Обрати цікаве відео зі списку результатів пошуку.
	2.Переглянути відео.
	3.Натиснути кнопку "Підписатися" під відео або на сторінці каналу.
•Очікуваний результат: Користувач успішно підписується на канал, кнопка змінюється на "Відписатися".

Тест-кейс: Сповіщення про новий контент
•Кроки:
	1.Підписатися на канал.
	2.Перейти до розділу "Підписки".
	3.Переглянути стрічку підписок.
•Очікуваний результат: У стрічці підписок відображаються нові відео від підписаного каналу.

Тест-кейси для User Story 2: Повернення одягу в інтернет-магазині
Тест-кейс: Пошук інформації про політику повернення
•Кроки: 
	1.Увійти на сайт інтернет-магазину. 
	2.Знайти розділ "Політика повернення". 
•Очікуваний результат: Відображається детальна інформація про політику повернення. 

Тест-кейс: Заповнення форми повернення товару
•Кроки: 
	1.Увійти в особистий кабінет на сайті. 
	2.Обрати розділ "Повернення товарів". 
	3.Заповнити форму повернення. 
•Очікуваний результат: Форма заповнена та відправлена успішно. 

Тест-кейс: Вказівка причини повернення та додавання фотографій
•Кроки: 
	1.Заповнити форму повернення. 
	2.Вказати причину повернення. 
	3.Додати фотографії товару (якщо потрібно). 
•Очікуваний результат: Причину вказано, фотографії додано. 

Тест-кейс: Отримання інструкцій для повернення товару
•Кроки: 
	1.Заповнити форму повернення. 
	2.Переглянути інструкції для відправки товару назад. 
•Очікуваний результат: Відображаються чіткі інструкції для відправки товару. 

Тест-кейс: Підтвердження прийому повернення та відстеження процесу
•Кроки: 
	1.Відправити товар назад згідно з інструкціями. 
	2.Отримати підтвердження про прийом повернення. 
	3.Відстежувати процес повернення через додаток. 
•Очікуваний результат: Користувач отримує підтвердження та може відстежувати процес повернення. 

Тест-кейси для User Story 3: Використання хмарного середовища
Тест-кейс: Завантаження файлів у хмарне сховище
•Кроки: 
	1.Увійти в додаток хмарного сховища. 
	2.Натиснути кнопку "Завантажити файл". 
	3.Вибрати файл на своєму пристрої. 
	4.Натиснути кнопку "Завантажити". 
•Очікуваний результат: Файл успішно завантажено у хмарне сховище. 

Тест-кейс: Спільне редагування документів в реальному часі
•Кроки: 
	1.Завантажити документ у хмарне сховище. 
	2.Надати доступ до документа іншому користувачеві. 
	3.Спільно редагувати документ. 
•Очікуваний результат: Зміни в документі відображаються в реальному часі для всіх користувачів. 

Тест-кейс: Встановлення рівнів доступу для файлів і папок
•Кроки: 
	1.Обрати файл або папку у хмарному сховищі. 
	2.Перейти до налаштувань доступу. 
	3.Встановити рівень доступу для інших користувачів. 
•Очікуваний результат: Рівні доступу успішно змінено. 

Тест-кейс: Пошук файлів за назвою або вмістом
•Кроки: 
	1.Відкрити додаток хмарного сховища. 
	2.Ввести назву файлу або ключові слова у рядок пошуку. 
	3.Натиснути кнопку "Пошук". 
•Очікуваний результат: Відображаються файли, що відповідають запиту. 

Тест-кейс: Сповіщення про зміни в документах
•Кроки: 
	1.Відкрити додаток хмарного сховища. 
	2.Дочекатися сповіщення про зміни в документах. 
	3.Переглянути історію змін файлу. 
•Очікуваний результат: Користувач отримує сповіщення про зміни та може переглянути історію. 

Тест-кейси для Use Case: Покупка продуктів на сайті супермаркету
Тест-кейс: Успішна покупка продуктів
•Кроки:
	1.Відкрити сайт супермаркету.
	2.Обрати потрібні продукти та додати їх у кошик.
	3.Перейти до кошика для перегляду обраних товарів.
	4.Підтвердити замовлення та перейти до оформлення покупки.
	5.Ввести дані для доставки та оплати.
	6.Підтвердити замовлення.
•Очікуваний результат: Замовлення успішно оформлено, користувач отримує підтвердження покупки.

Тест-кейс: Покупка продуктів без інтернет-з'єднання
•Кроки:
	1.Відкрити сайт супермаркету без підключення до інтернету.
	2.Спробувати додати продукти у кошик та оформити замовлення.
•Очікуваний результат: Сайт повідомляє про відсутність з'єднання з інтернетом і пропонує спробувати пізніше.

Тест-кейс: Помилка при обробці оплати
•Кроки:
	1.Відкрити сайт супермаркету.
	2.Обрати продукти та додати їх у кошик.
	3.Перейти до кошика для перегляду обраних товарів.
	4.Підтвердити замовлення та перейти до оформлення покупки.
	5.Ввести неправильні дані для оплати.
	6.Підтвердити замовлення.
•Очікуваний результат: Система повідомляє про помилку оплати, замовлення не оформлено.

Тест-кейси для Use Case: Переклад тексту за допомогою онлайн перекладача
Тест-кейс: Успішний переклад тексту
•Кроки:
	1.Відкрити онлайн перекладач.
	2.Ввести текст для перекладу.
	3.Вибрати мову для перекладу.
	4.Натиснути кнопку "Перекласти".
•Очікуваний результат: Система успішно перекладає текст, користувач отримує перекладений текст на екрані.

Тест-кейс: Переклад тексту без інтернет-з'єднання
•Кроки:
	1.Відкрити онлайн перекладач без підключення до інтернету.
	2.Ввести текст для перекладу.
	3.Натиснути кнопку "Перекласти".
•Очікуваний результат: Система повідомляє про відсутність з'єднання з інтернетом і пропонує спробувати пізніше.

Тест-кейс: Помилка при введенні тексту
•Кроки:
	1.Відкрити онлайн перекладач.
	2.Ввести текст, який містить недопустимі символи або не підтримується для перекладу.
	3.Натиснути кнопку "Перекласти".
•Очікуваний результат: Система повідомляє про помилку та інформує користувача, що введений текст не може бути перекладений.
